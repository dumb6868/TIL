# 네트워크
둘 이상의 컴퓨터나 장치들이 그물망 형태로 연결되어, 데이터와 자원을 공유하고 통신할 수 있는 인프라이다.

#### -프로토콜
컴퓨터나 원거리 통신 장비 사이에서 데이터를 원활하게 주고받기 위해 약속한 규칙, 절차, 형식의 체계이다.
*예:https://

#### -알고리즘
문제를 해결하기 위한 절차나 방법이다.

## TCP (Transmission Control Protocol)
데이터를 정확하고 순서대로 전달하는 프로토콜이다.
### 특징
- 신뢰성, 순서 보장
- 중간에 데이터가 빠지면 재전송한다.
- 통신 전에 연결부터 맺는다.(연결지향)

## UDP(User Datagram Protocol)
빠르게 보내지만, 신뢰성은 보장하지 않는 프로토콜이다.
### 특징
- 확인 없이 그냥 보내서 빠르다.
- 순서 보장이 업고 중간에 데이터가 날아갈 수 있다.
- 연결 없이 발 전송한다.(비연결)

## IP주소(Internet Protocol Address)
인터넷에 연결된 컴퓨터, 스마트폰 등 기기들이 서로 통신하기 위해 할당받는 고유한 식별 번호로, '인터넷상의 주소' 역할을 한다.

## 도메인(Domain)
복잡한 숫자 형태의 IP 주소를 사람이 기억하기 쉽게 문자로 표현한 인터넷상의 고유한 주소이다.
간단히 말하면 서버의 위치(이름)이다.
*예:www,google.com

## URL(Uniform Resource Locator)
웹 페이지, 이미지, 문서 등 인터넷상의 고유한 리소스 위치를 나타내는 주소이다.
간단히 말하면 어디, 무엇을 요청할지까지 포함한 전체 주소이다.
