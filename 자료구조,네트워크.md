# 자료구조
데이터를 구조적으로 표현하는 방식이다.

## Linked list(연결 리스트)
임의 접근(Random Access)과 임의 삽입/삭제가 가능한 선형 자료구조이다. 
### 특징
-동적인 데이터 추가/삭제에 유리하다.
-메모리에 연속적으로 저장되지 않는다.
-탐색 속도가 느리다.
-주소는 랜덤하지만 포인터(다음 주소)로 논리적 순서가 유지된다.

### Array List(배열 리스트)
연속된 메모리 공간에 데이터를 저장하고, 인덱스를 통해 빠르게 접근할 수 있는 선형 자료구조이다.
### 특징
-연속된 메모리에 저장된다
-크기가 동적으로 변한다
-인덱스로 빠르게 접근한다

## Queue(큐)
선입선출(FIFO, First In First Out) 방식의 선형 자료구조로, 한쪽 끝에서 삽입하고 반대쪽 끝에서 삭제한다.
### 특징 
-먼저 들어온 것이 먼저 나간다.
-한쪽에서 삽입, 반대쪽에서 삭제한다.

## Stack(스택)
후입선출(LIFO, Last In First Out) 방식의 선형 자료구조로, 한쪽 끝에서만 삽입과 삭제가 이루어진다.
### 특징
-한쪽 끝에서만 삽입/삭제한다.
-구조가 단순하다

## Hash Map(해시맵)
해시맵은 키(key)를 해시 함수로 변환해 저장 위치를 계산하고, 그 위치에서 값을 빠르게 찾는 자료구조이다.
키(key)런 데이터를 식별하고 검색하는 고유한 고유 ID이다.
### 특징
-순서를 보장하지 않는다.
-키는 중복될 수 없다.
-탐색이 빠르다.

# 네트워크
둘 이상의 컴퓨터나 장치들이 그물망 형태로 연결되어, 데이터와 자원을 공유하고 통신할 수 있는 인프라이다.

#### -프로토콜
컴퓨터나 원거리 통신 장비 사이에서 데이터를 원활하게 주고받기 위해 약속한 규칙, 절차, 형식의 체계이다.
#### -알고리즘
문제를 해결하기 위한 절차나 방법이다.

## TCP (Transmission Control Protocol)
데이터를 정확하고 순서대로 전달하는 프로토콜이다.
### 특징
-신뢰성, 순서 보장
-중간에 데이터가 빠지면 재전송한다.
-통신 전에 연결부터 맺는다.(연결지향)

## UDP(User Datagram Protocol)
빠르게 보내지만, 신뢰성은 보장하지 않는 프로토콜이다.
### 특징
-확인 없이 그냥 보내서 빠르다.
-순서 보장이 업고 중간에 데이터가 날아갈 수 있다.
-연결 없이 발 전송한다.(비연결)
