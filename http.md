## HTTP
HTTP는 클라이언트와 서버가 데이터를 주고받기 위한 통신 규칙이다.  
웹에서 요청과 응답 방식으로 동작한다.

### 특징
- Stateless: 각 요청은 서로 독립적이며 이전 상태를 기억하지 않는다.
- 텍스트 기반 프로토콜이라 구조를 사람이 읽을 수 있다.
- 주로 TCP 기반 위에서 동작한다.

## HTTP method
HTTP Method는 클라이언트가 서버에 어떤 동작을 요청하는지 나타낸다.

### 특징
- 같은 URL이라도 method에 따라 수행되는 동작이 달라진다.
- 자원의 상태를 어떻게 변경할지 정의하는 역할을 한다.

### 주요 메서드
- GET: 데이터 조회
- POST: 데이터 생성
- PUT: 데이터 전체 수정
- PATCH: 데이터 일부 수정
- DELETE: 데이터 삭제


## API
API는 서로 다른 프로그램이 통신할 수 있도록 해주는 인터페이스이다.  
보통 HTTP를 기반으로 요청과 응답 형태로 데이터를 주고받는다.

### 특징
- 요청과 응답 구조로 동작한다.
- JSON 형식을 많이 사용한다.
- 프론트엔드와 백엔드를 연결하는 역할을 한다.
