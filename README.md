# TIL
## 객체지향 프로그램이란
객체 지향 프로그래밍(Object-Oriented Programming, OOP)은 프로그램을 객체(Object) 중심으로 설계하고 구현하는 프로그래밍 패러다임이다. 쉽게 설명하면 물건을 객체로 만들어 프로그램을 만드는 방법이다.
예를 들어 게임에서 몬스터를 생각해 보면 몬스터는 이름, 체력, 공격력 등의 **정보**와  
공격하기, 이동하기, 죽기 등의 **행동**을 가진다.
이처럼 정보와 행동을 함께 가지는 하나의 몬스터를 **객체**라고 할 수 있다.

### 특징
#### 1.캡슐화
캡슐화란 데이터와 메서드를 하나의 객체로 묶고 외부에서 직접 접근하지 못하도록 보호하는 것이다.
쉽게 말하면 중요한 것을 안에 숨기는 것.
리모컨엔 회로, 배선, 전자 부품 등 복잡한 것들이 들어있지만 우린 그걸 몰라도 버튼만 누르면 TV가 켜진다.

#### 2.상속
상속이란 기존 클래스의 속성과 메서드를 다른 클래스가 물려받는 것이다.
쉽게 말하면 부모의 특징을 자식이 물려받는 것.
인간은 동물이기에 동물의 먹기, 자기 등의 특징을 물려받는다. 

#### 3. 다형성
다형성이란 다같은 메서드 이름이지만 다른 동작을 수행할 수 있는 것이다.
쉽게 말하면 같은 행동인데 결과가 다른 것. 
예를 들자면 나와 누군가가 소리를 낼 때 소리를 낸다는 ***행동***은 같지만 ***소리***는 다르다.

#### 4. 추상화 
추상화란 불필요한 세부 구현은 숨기고 핵심적인 기능만 보여주는 것이다.
쉽게 말하면 중요한 것만 보여주는 것.
캡슐화와 반대되는 느낌이다.

### 장점
#### 코드 재사용성
***상속***을 통해 기존 클래스의 기능을 재사용할 수 있다.

#### 유지 보수 용이
객체 단위로 코드가 나뉘어 있기 때문에 문제가 발생한 부분만 수정하면 된다.

#### 모델링
프로그램 설계가 직관적이다.

### 단점
#### 설계
객체 간의 관계를 먼저 설계해야 하기 때문에 초기 설계 과정이 복잡할 수 있다.

#### 실행 속도
절차지향 프로그래밍에 비해 객체 생성, 메서드 호출 등이 많아져 성능이 떨어질 수 있다.

## 느낀점
객체 지향 프로그래밍을 처음 알게 되어 기뻤다. 
처음에는 개념이 어렵지만 구조적으로 프로그램을 설계할 수 있다는
장점이 있다고 느꼈다.
